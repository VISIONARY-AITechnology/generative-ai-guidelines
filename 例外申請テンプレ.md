## 6. リスク評価

### 6-1. リスク分析

| リスク項目 | 発生可能性 | 影響度 | リスクレベル |
|-----------|:----------:|:------:|:------------:|
| 情報漏洩 | ☐低 ☐中 ☐高 | ☐低 ☐中 ☐高 | |
| ハルシネーション | ☐低 ☐中 ☐高 | ☐低 ☐中 ☐高 | |
| 著作権侵害 | ☐低 ☐中 ☐高 | ☐低 ☐中 ☐高 | |
| 契約違反 | ☐低 ☐中 ☐高 | ☐低 ☐中 ☐高 | |
| セキュリティ脆弱性 | ☐低 ☐中 ☐高 | ☐低 ☐中 ☐高 | |

**リスクレベル計算**: 発生可能性 × 影響度
- 低×低/低×中 = 低リスク
- 中×中/低×高/高×低 = 中リスク
- 中×高/高×中/高×高 = 高リスク

## 付録：申請書記入例

### 【記入例1】新規ツール導入

```
■ 申請カテゴリ: 新規ツール導入
■ ツール名: Cursor（AIコードエディタ）
■ 利用目的: 開発効率の向上、コーディング補助
■ 入力データ: 分類1（公開情報）、分類2（社内コード）
■ リスク軽減策:
  - 顧客コードは入力しない運用ルールを設定
  - 学習オフ設定を有効化
  - 利用者への教育を実施
```

### 【記入例2】区分3利用（受託開発）

```
■ 申請カテゴリ: 区分3利用（新規）
■ プロジェクト: A社向け問い合わせ対応システム開発
■ 使用ツール: Azure OpenAI Service
■ 入力データ: 分類5（顧客機密：A社のFAQデータ）
■ 顧客承認: 契約書別紙でAI利用を許可済み
■ 責任分界:
  - 当社：システム開発・納品時点の品質保証
  - 顧客：運用環境でのデータ管理・出力内容の最終判断
```
