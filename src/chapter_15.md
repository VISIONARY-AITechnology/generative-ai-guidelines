### 付録B FAQ

| 質問 | 回答 |
|-----|------|
| ChatGPTを業務で使ってよいか？ | 許可ツールリストに掲載されていれば可。データ分類に従った入力制限あり。 |
| 顧客のデータを入力してよいか？ | 顧客契約で許可されており、かつオプトアウト設定済みのツールであれば、区分2として申請後に可。 |
| AI生成物は著作権で保護されるか？ | 人間の創作的関与がない場合は保護されない可能性あり。法務に相談。 |
| インシデントを起こしたらどうなるか？ | 故意でない限り処分はなし。速やかな報告が重要。隠蔽は処分対象。 |

### 付録C ツール別設定ガイダンス

#### C.1 Azure OpenAI Service
| 設定項目 | 推奨設定 |
|---------|---------|
| データの学習利用 | 無効（デフォルトで無効） |
| コンテンツフィルタリング | 有効 |
| 監査ログ | Azure Monitor連携有効 |

#### C.2 AWS Bedrock
| 設定項目 | 推奨設定 |
|---------|---------|
| モデル呼び出しログ | CloudWatch Logs有効 |
| IAMポリシー | 最小権限の原則適用 |
| VPCエンドポイント | 利用推奨 |

#### C.3 ChatGPT Enterprise
| 設定項目 | 推奨設定 |
|---------|---------|
| チャット履歴の学習 | 無効（Enterpriseはデフォルト無効） |
| SSO | 有効 |
| データエクスポート | 管理者のみに制限 |

#### C.4 Claude Enterprise
| 設定項目 | 推奨設定 |
|---------|---------|
| データの学習利用 | 無効（Enterpriseはデフォルト無効） |
| SSO | 有効 |
| 監査ログ | 有効 |

#### C.5 Google Gemini
| 設定項目 | 推奨設定 |
|---------|---------|
| Workspace連携 | 組織ポリシーに従い設定 |
| データの学習利用 | 組織設定で無効化 |

#### C.6 自社構築環境
| 設定項目 | 推奨設定 |
|---------|---------|
| ネットワーク | 社内ネットワーク内またはVPN経由のみ |
| 認証 | 社内ID基盤連携、MFA必須 |
| ログ | 全入出力ログの保存 |
| バックアップ | 日次バックアップ |
