# 生成AI利用チェックリスト

**文書番号**: GL-AI-002
**版数**: 1.0
**最終更新日**: 2025年1月
**目的**: 5分以内で生成AI利用可否を判断するための簡易チェックリスト

---

## 使い方

1. **セクションA** でデータ分類を特定
2. **セクションB** で利用区分を特定
3. **セクションC** の判定マトリクスで利用可否を確認
4. **セクションD** で必要な手続きを実行

---

## セクションA: 入力データ分類チェック

入力しようとしているデータについて、該当するものをチェックしてください。

### A-1. 個人情報チェック

| チェック | 質問 | 該当する場合 |
|:-------:|------|-------------|
| ☐ | マイナンバー、健康診断結果、犯罪歴を含む | → **分類6: 特定個人情報** 🔴 |
| ☐ | 氏名+連絡先など個人を特定できる情報を含む | → **分類4: 個人情報** 🟠 |

### A-2. 顧客情報チェック

| チェック | 質問 | 該当する場合 |
|:-------:|------|-------------|
| ☐ | 顧客から預かったデータ（ソースコード、設計書等） | → **分類5: 顧客機密** 🟠 |
| ☐ | 顧客との契約でAI利用が明示的に許可されている | → 許可範囲内で利用可 |

### A-3. 社内情報チェック

| チェック | 質問 | 該当する場合 |
|:-------:|------|-------------|
| ☐ | 経営戦略、未公開財務情報、M&A情報を含む | → **分類3: 機密情報** 🟠 |
| ☐ | 社外に出ると問題だが、上記ほど機密性は高くない | → **分類2: 社外秘** 🟡 |
| ☐ | 公開されている情報、一般的な技術情報のみ | → **分類1: 公開情報** 🟢 |

### 【判定結果】入力データの分類

```
分類1: 公開情報    🟢 → セクションBへ
分類2: 社外秘      🟡 → セクションBへ
分類3: 機密情報    🟠 → セクションBへ（承認必要）
分類4: 個人情報    🟠 → 匿名化を検討、セクションBへ
分類5: 顧客機密    🟠 → 契約確認必須、セクションBへ
分類6: 特定個人情報 🔴 → AI利用不可、手動処理へ
```

---

## セクションB: 利用区分チェック

利用目的について、該当するものをチェックしてください。

### B-1. 基本区分の判定

| チェック | 質問 | 該当する場合 |
|:-------:|------|-------------|
| ☐ | 自社の業務効率化（コード補助、文書作成、調査等） | → **区分1: 社内業務** |
| ☐ | 受託開発・SESで顧客データを扱う | → **区分2: 顧客データ取扱い** |
| ☐ | AI機能を含む製品・サービスを開発する | → **区分3: 顧客向け開発** |
| ☐ | SESとして顧客環境で生成AIを利用する | → **区分3: 顧客環境利用** |
| ☐ | AI導入支援のコンサルティングを行う | → **区分3: AI導入支援** |

### 【判定結果】利用区分

```
区分1: 社内業務利用     → 部門長承認で利用可
区分2: 顧客データ取扱い → PJ責任者承認、契約確認必須
区分3: 顧客向け開発等   → AI審査会議または簡易審査が必要
```

---

## セクションC: 利用可否判定マトリクス

データ分類（縦）× 利用区分（横）で判定してください。

### C-1. 判定表

| データ分類 | 区分1: 社内業務 | 区分2: 顧客データ | 区分3: 顧客向け |
|-----------|:-------------:|:---------------:|:-------------:|
| 分類1: 公開 | ✅ 利用可 | ✅ 利用可 | ✅ 利用可 |
| 分類2: 社外秘 | ✅ 利用可 | ⚠️ 注意 | ⚠️ 審査 |
| 分類3: 機密 | ⚠️ 承認制 | ⚠️ 承認制 | 🔶 審査必須 |
| 分類4: 個人情報 | ⚠️ 匿名化 | ⚠️ 匿名化+契約 | 🔶 審査必須 |
| 分類5: 顧客機密 | - | ⚠️ 契約確認 | 🔶 審査必須 |
| 分類6: 特定個人情報 | ❌ 禁止 | ❌ 禁止 | ❌ 禁止 |

### C-2. 凡例

| 記号 | 意味 | 次のアクション |
|:---:|------|---------------|
| ✅ | 利用可 | セクションDの基本チェックへ |
| ⚠️ | 条件付き利用可 | 記載条件を満たしてセクションDへ |
| 🔶 | 審査必要 | セクションEの審査申請へ |
| ❌ | 利用禁止 | AI以外の手段を検討 |

---

## セクションD: 利用前チェックリスト

利用可能と判定された場合、以下を確認してください。

### D-1. 必須チェック（全員）

| ☐ | チェック項目 | 確認方法 |
|:-:|-------------|---------|
| ☐ | 使用ツールは承認済みリストに載っている | 社内ポータル参照 |
| ☐ | 学習オフ（オプトアウト）設定が有効 | ツール設定画面 |
| ☐ | ログ記録が有効になっている | ツール設定画面 |
| ☐ | 入力内容に不要な機密情報が混入していない | 目視確認 |

### D-2. 追加チェック（区分2・3）

| ☐ | チェック項目 | 確認方法 |
|:-:|-------------|---------|
| ☐ | 顧客との契約でAI利用が許可されている | 契約書・覚書 |
| ☐ | 顧客への報告・承認が完了している | メール・議事録 |
| ☐ | プロジェクト責任者の承認を得ている | 承認記録 |
| ☐ | 個人情報は匿名化・マスキング済み | 処理記録 |

### D-3. 追加チェック（区分3のみ）

| ☐ | チェック項目 | 確認方法 |
|:-:|-------------|---------|
| ☐ | 責任分界が契約書に明記されている | 契約書 |
| ☐ | 品質基準（ハルシネーション対策等）が定義済み | 設計書 |
| ☐ | Human-in-the-loop の設計がされている | 設計書 |
| ☐ | セキュリティ要件が定義・実装済み | セキュリティ仕様書 |

## セクションE: 審査が必要な場合

🔶マークがついた場合、以下の手順で審査を申請してください。

### E-1. 審査種別の判定

| 条件 | 審査種別 | 審査者 | 所要期間 |
|-----|---------|--------|---------|
| 新規ツール導入 | 正式審査 | AI審査会議 | 1-2週間 |
| 区分3（初回案件） | 正式審査 | AI審査会議 | 1-2週間 |
| 区分3（類似実績あり） | 簡易審査 | AI推進責任者 | 2-3営業日 |
| 機密データ入力（単発） | 簡易審査 | AI推進責任者 | 2-3営業日 |
| 外部API連携追加 | 簡易審査 | AI推進責任者 | 2-3営業日 |

### E-2. 審査申請手順

1. [例外申請テンプレート](./exception.md) をダウンロード
2. 必要事項を記入
3. 一次承認者（部門長）の承認を取得
4. AI審査会議事務局へ提出
5. 審査結果を待つ

## セクションG: 利用後チェックリスト

### G-1. 出力確認

| ☐ | チェック項目 |
|:-:|-------------|
| ☐ | 出力内容に機密情報が含まれていない |
| ☐ | 出力内容が事実と異なる記述（ハルシネーション）がない |
| ☐ | 著作権侵害の恐れがある表現がない |
| ☐ | 差別的・不適切な表現がない |

### G-2. 保存・廃棄判断

| 条件 | 対応 |
|-----|------|
| 一時的な利用で保存不要 | 即時削除 |
| 社内業務で再利用する | 適切なフォルダに保存 |
| 顧客への納品物に含める | 納品手順に従う |
| 契約終了後 | 契約に基づき廃棄 |

### G-3. インシデント発生時

以下に該当する場合、直ちに報告してください：

| 事象 | 報告先 | 対応 |
|-----|-------|------|
| 機密情報を誤って入力した | 情報セキュリティ責任者 | インシデント対応手順 |
| 個人情報を入力してしまった | 情報セキュリティ責任者 | 個人情報漏洩対応 |
| 出力に顧客の機密情報が含まれていた | PJ責任者+セキュリティ | 顧客への報告検討 |
| 不適切な出力が外部に流出した | AI推進責任者+法務 | 影響評価・対応 |

## クイックチェック（最短版）

**以下すべてYesなら利用可能です：**

1. ☐ 特定個人情報（マイナンバー等）を含まない
2. ☐ 顧客機密を入力する場合、契約で許可されている
3. ☐ 使用ツールは承認済みリストに載っている
4. ☐ 学習オフ設定が有効である
5. ☐ 必要な承認を取得済み

**1つでもNoがある場合：**
- セクションA〜Eに戻って詳細確認
- 判断に迷う場合はAI推進責任者に相談
