## 1. 全体フロー概要

生成AI利用時の判断プロセス全体像を示します。

```mermaid
flowchart TB
    Start([生成AIを利用したい]) --> A[1. 入力データの分類]
    A --> B[2. 利用区分の判定]
    B --> C[3. ツール選定・確認]
    C --> D[4. 審査要否の判定]
    D --> E{審査が必要?}
    E -->|Yes| F[審査申請・承認待ち]
    F --> G{承認された?}
    G -->|Yes| H[5. 実行・ログ記録]
    G -->|No| I[代替手段を検討]
    I --> A
    E -->|No| H
    H --> J[6. 保存・廃棄判断]
    J --> End([完了])

    style Start fill:#e1f5fe
    style End fill:#e8f5e9
    style F fill:#fff3e0
    style I fill:#ffebee
```

## 2. 入力データ分類フロー

**根拠**: IPA生成AIガイドライン p.40-41「入力情報の制限」

入力しようとしているデータの機密レベルを判定します。

```mermaid
flowchart TB
    Start([入力データの確認]) --> Q1{個人を特定できる<br/>情報を含む?}

    Q1 -->|Yes| Q1a{マイナンバー・<br/>健康情報・犯罪歴?}
    Q1a -->|Yes| C6[分類6: 特定個人情報<br/>🔴 原則入力禁止]
    Q1a -->|No| C4[分類4: 個人情報<br/>🟠 厳重管理]

    Q1 -->|No| Q2{顧客から預かった<br/>情報?}

    Q2 -->|Yes| Q2a{顧客との契約で<br/>AI利用が許可?}
    Q2a -->|Yes| C5[分類5: 顧客機密<br/>🟠 契約範囲内で利用]
    Q2a -->|No| C5b[分類5: 顧客機密<br/>🔴 入力禁止]

    Q2 -->|No| Q3{社外に出ると<br/>損害が生じる?}

    Q3 -->|Yes| Q3a{経営戦略・<br/>未公開財務・<br/>技術機密?}
    Q3a -->|Yes| C3[分類3: 機密情報<br/>🟠 承認制]
    Q3a -->|No| C2[分類2: 社外秘<br/>🟡 注意して利用]

    Q3 -->|No| C1[分類1: 公開情報<br/>🟢 利用可]

    style C1 fill:#c8e6c9
    style C2 fill:#fff9c4
    style C3 fill:#ffe0b2
    style C4 fill:#ffccbc
    style C5 fill:#ffe0b2
    style C5b fill:#ffcdd2
    style C6 fill:#ffcdd2
```

### データ分類早見表

| 分類 | 例 | 入力可否 | 必要な措置 |
|-----|-----|---------|-----------|
| 分類1: 公開 | Webサイト掲載情報、プレスリリース | ✅ 可 | なし |
| 分類2: 社外秘 | 社内手順書、会議メモ | ⚠️ 条件付 | 学習オフ確認 |
| 分類3: 機密 | 経営計画、未発表製品情報 | ⚠️ 承認制 | 部長承認 |
| 分類4: 個人情報 | 氏名+連絡先、社員情報 | ⚠️ 厳重管理 | 匿名化推奨 |
| 分類5: 顧客機密 | 顧客のソースコード、設計書 | ⚠️ 契約確認 | 顧客許可必須 |
| 分類6: 特定個人情報 | マイナンバー、健康診断結果 | ❌ 禁止 | 代替手段 |

## 3. 利用区分判定フロー

**根拠**: 総務省ガイドライン p.30-39「各主体の取組事項」

利用目的と対象に基づいて利用区分を判定します。

```mermaid
flowchart TB
    Start([利用目的の確認]) --> Q1{誰のための<br/>作業?}

    Q1 -->|自社のため| Q2{顧客データを<br/>使用する?}
    Q2 -->|No| K1[区分1: 社内業務利用<br/>📋 部門長承認]
    Q2 -->|Yes| K2[区分2: 顧客データ取扱い<br/>📋 プロジェクト責任者承認]

    Q1 -->|顧客のため| Q3{作業形態は?}

    Q3 -->|受託開発| Q4{AI機能を<br/>成果物に含む?}
    Q4 -->|Yes| K3a[区分3-A: AI機能開発<br/>📋 AI審査会議]
    Q4 -->|No| K2

    Q3 -->|SES| Q5{顧客環境で<br/>AI利用?}
    Q5 -->|Yes| K3b[区分3-B: 顧客環境利用<br/>📋 顧客承認+部長承認]
    Q5 -->|No| K1

    Q3 -->|コンサルティング| Q6{AI導入支援?}
    Q6 -->|Yes| K3c[区分3-C: AI導入支援<br/>📋 AI推進責任者承認]
    Q6 -->|No| K1

    style K1 fill:#c8e6c9
    style K2 fill:#fff9c4
    style K3a fill:#ffe0b2
    style K3b fill:#ffe0b2
    style K3c fill:#ffe0b2
```

### 利用区分別の要件サマリ

| 区分 | 主な用途 | 承認者 | 主要要件 |
|-----|---------|--------|---------|
| 区分1 | コード補助、文書作成、調査 | 部門長 | 学習オフ設定確認 |
| 区分2 | 顧客データ分析、テストデータ生成 | PJ責任者 | 契約確認、匿名化 |
| 区分3-A | AI機能を含む受託開発 | AI審査会議 | 責任分界、品質基準 |
| 区分3-B | SESで顧客環境のAI利用 | 顧客+部長 | 顧客規程遵守 |
| 区分3-C | AIコンサルティング | AI推進責任者 | 助言範囲明確化 |

## 4. ツール選定・確認フロー

**根拠**: IPA生成AIガイドライン p.31-33「システム構築環境」

適切なツールを選定し、設定を確認します。

```mermaid
flowchart TB
    Start([ツール選定]) --> Q1{データ分類は?}

    Q1 -->|分類1-2| T1{承認済みツール<br/>リストに載っている?}
    T1 -->|Yes| C1[設定確認へ]
    T1 -->|No| A1[新規ツール申請<br/>→審査必要]

    Q1 -->|分類3-5| T2{企業向けプラン<br/>or オンプレ?}
    T2 -->|Yes| C2[設定確認へ]
    T2 -->|No| R1[❌ 利用不可<br/>企業向けプランを検討]

    Q1 -->|分類6| R2[❌ AI利用不可<br/>手動処理へ]

    C1 --> S1{学習オフ設定<br/>確認済み?}
    C2 --> S1

    S1 -->|Yes| S2{ログ取得設定<br/>確認済み?}
    S1 -->|No| S1a[設定を有効化]
    S1a --> S2

    S2 -->|Yes| S3{データ保持期間<br/>確認済み?}
    S2 -->|No| S2a[設定を有効化]
    S2a --> S3

    S3 -->|Yes| OK([✅ 利用可能])
    S3 -->|No| S3a[保持期間を確認・設定]
    S3a --> OK

    style OK fill:#c8e6c9
    style R1 fill:#ffcdd2
    style R2 fill:#ffcdd2
    style A1 fill:#fff9c4
```

### 承認済みツールと設定チェック

| ツール | 対応分類 | 必須設定 | 確認方法 |
|-------|---------|---------|---------|
| Azure OpenAI | 1-5 | データオプトアウト済み | Azure Portal |
| AWS Bedrock | 1-5 | モデル呼び出しログ有効 | CloudWatch |
| ChatGPT Enterprise | 1-4 | 学習オフ（デフォルト） | 管理コンソール |
| Claude Enterprise | 1-4 | 学習オフ（デフォルト） | 管理コンソール |
| Gemini for Google Workspace | 1-3 | 管理者設定確認 | Google Admin |
| 自社構築（オンプレ） | 1-5 | 社内セキュリティ基準 | インフラチーム |

## 5. 審査要否判定フロー

**根拠**: IPA生成AIガイドライン p.52-60「リスクアセスメント」

審査申請が必要かどうかを判定します。

```mermaid
flowchart TB
    Start([審査要否の判定]) --> Q1{新規ツール・<br/>モデルの導入?}

    Q1 -->|Yes| R1[🔶 審査必要<br/>AI審査会議へ申請]

    Q1 -->|No| Q2{区分3の<br/>利用?}

    Q2 -->|Yes| Q2a{過去に同様案件で<br/>承認実績あり?}
    Q2a -->|Yes| R2[🔷 簡易審査<br/>AI推進責任者承認]
    Q2a -->|No| R1

    Q2 -->|No| Q3{分類3以上の<br/>データ入力?}

    Q3 -->|Yes| Q3a{匿名化・<br/>マスキング済み?}
    Q3a -->|Yes| R3[✅ 審査不要<br/>通常承認フローへ]
    Q3a -->|No| R2

    Q3 -->|No| Q4{外部API・<br/>ツール連携?}

    Q4 -->|Yes| R2
    Q4 -->|No| R3

    style R1 fill:#ffe0b2
    style R2 fill:#fff9c4
    style R3 fill:#c8e6c9
```

### 審査レベル別の対応

| 審査レベル | トリガー | 審査者 | 所要期間目安 |
|-----------|---------|--------|------------|
| 審査不要 | 区分1-2、分類1-2、既存ツール | 部門長 | 即日 |
| 簡易審査 | 区分3（実績あり）、分類3データ | AI推進責任者 | 2-3営業日 |
| 正式審査 | 新規ツール、区分3（新規）、外部連携 | AI審査会議 | 1-2週間 |

## 6. 実行・ログ記録フロー

**根拠**: 総務省ガイドライン p.18-20「透明性」、IPA p.43-46「透明性確保」

実際の利用時のログ記録と確認事項を示します。

```mermaid
flowchart TB
    Start([実行開始]) --> P1[入力内容の最終確認]

    P1 --> P2{機密情報が<br/>含まれていない?}
    P2 -->|No| P2a[該当部分を削除・マスキング]
    P2a --> P1
    P2 -->|Yes| P3[プロンプト実行]

    P3 --> L1[📝 ログ自動記録]

    L1 --> L2[記録項目:<br/>・実行日時<br/>・利用者ID<br/>・使用ツール<br/>・利用区分<br/>・プロンプト概要]

    L2 --> O1[出力結果の確認]

    O1 --> O2{出力に問題<br/>がある?}

    O2 -->|Yes| O2a{問題の種類}
    O2a -->|機密情報漏洩| I1[🚨 インシデント報告<br/>→12章参照]
    O2a -->|不正確な情報| O3[出力を破棄<br/>再実行を検討]
    O2a -->|著作権懸念| O4[法務確認<br/>→10章参照]

    O2 -->|No| O5[出力結果を利用]
    O3 --> P1

    O5 --> End([保存・廃棄判断へ])

    style O4 fill:#fff9c4

```

## 7. 保存・廃棄判断フロー

**根拠**: IPA生成AIガイドライン p.46-49「運用ルール」

生成物の保存・廃棄を判断します。

```mermaid
flowchart TB
    Start([保存・廃棄判断]) --> Q1{生成物を<br/>保存する必要がある?}

    Q1 -->|No| D1[即時廃棄<br/>ローカル削除]

    Q1 -->|Yes| Q2{保存先は?}

    Q2 -->|社内システム| S1{データ分類に<br/>対応した保管場所?}
    S1 -->|Yes| S2[保存実行<br/>アクセス権限設定]
    S1 -->|No| S1a[適切な保管場所を選択]
    S1a --> S2

    Q2 -->|顧客への納品| S3{納品物として<br/>契約に含まれる?}
    S3 -->|Yes| S4[納品手順に従う<br/>社内コピーの扱いを確認]
    S3 -->|No| S5[顧客と協議<br/>追加契約を検討]

    S2 --> R1[保持期間の設定]
    S4 --> R1

    R1 --> R2{区分と分類に<br/>応じた保持期間}

    R2 --> R3[定期レビュー<br/>スケジュール設定]

    R3 --> End([完了])
    D1 --> End

    style D1 fill:#e1f5fe
    style S2 fill:#c8e6c9
    style S4 fill:#c8e6c9
    style S5 fill:#fff9c4
```

### 保持期間ガイドライン

| データ分類 | 利用区分 | 推奨保持期間 | 廃棄方法 |
|-----------|---------|------------|---------|
| 分類1-2 | 区分1 | 必要期間のみ | 通常削除 |
| 分類3 | 区分1-2 | 1年 | 完全削除 |
| 分類4-5 | 区分2-3 | プロジェクト終了+1年 | 完全削除+記録 |
| - | 区分3（受託） | 契約に基づく | 契約に基づく |

## 8. クイックリファレンス

### 5分で判断するための早見表

```
【STEP 1】入力データは何？
  └─ 個人情報あり → 匿名化 or 入力禁止
  └─ 顧客データ → 契約確認必須
  └─ 機密情報 → 承認必要
  └─ 上記以外 → STEP 2へ

【STEP 2】誰のため？
  └─ 自社業務 → 区分1（部門長承認）
  └─ 顧客データ使用 → 区分2（PJ責任者承認）
  └─ 顧客向け開発・SES → 区分3（審査会議）

【STEP 3】ツールは承認済み？
  └─ 承認済み → 設定確認して利用
  └─ 未承認 → 新規ツール申請

【STEP 4】審査が必要？
  └─ 新規ツール・区分3（初回） → 正式審査
  └─ 区分3（実績あり）・機密データ → 簡易審査
  └─ それ以外 → 審査不要

【STEP 5】利用後
  └─ ログ記録を確認
  └─ 出力を検証
  └─ 保存/廃棄を判断
```

### 困ったときの連絡先

| 状況 | 連絡先 | 対応 |
|-----|-------|------|
| 判断に迷う | AI推進責任者 | 相談・助言 |
| 緊急インシデント | 情報セキュリティ責任者 | 即時対応 |
| ツール追加希望 | AI審査会議事務局 | 申請受付 |
| 顧客対応で困った | 営業担当+法務 | 契約確認 |
