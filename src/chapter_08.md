# 第8章 セキュリティ要件

## 8.1 認証・認可

| 要件 | 内容 | 適用区分 |
|-----|------|---------|
| SSO（シングルサインオン） | 社内ID基盤との連携推奨 | 全区分 |
| MFA（多要素認証） | 区分2・3では必須、区分1は推奨 | 区分2/3必須 |
| 権限の定期レビュー | 四半期ごとの権限棚卸し | 全区分 |

## 8.2 権限管理

- 最小権限原則の適用
- 職務分掌に基づく権限設計
- 特権アカウントの厳格管理

> **根拠**: IPA p.62-63

## 8.3 ログ管理

| ログ種別 | 記録内容 | 保存期間 |
|---------|---------|---------|
| アクセスログ | ユーザーID、日時、接続元IP | 1年間 |
| 操作ログ | 入力プロンプト、出力内容 | 1年間（区分2/3は3年間） |
| エラーログ | エラー内容、発生日時 | 1年間 |
| 監査ログ | 設定変更、権限変更 | 3年間 |

> **根拠**: 総務省 p.18-19、IPA p.64

## 8.4 データ保護

| 対策 | 内容 |
|-----|------|
| 通信暗号化 | TLS 1.2以上 |
| 保管時暗号化 | AES-256等 |
| DLP検討 | 機密データの外部送信検知・遮断（区分2/3で検討） |
| バックアップ | 定期バックアップと復旧テスト |

## 8.5 多層防御

> **根拠**: IPA p.61-64

### 8.5.1 入口対策
- プロンプト文の検証（禁止文字列のフィルタリング）
- ガードレールの設置（入力ポリシー違反の検知）
- 入力ログの保存

### 8.5.2 内部対策
- 最小権限でのプロセス実行
- 防御用プロンプトテンプレートの実装
- コンテナ・サンドボックス環境の利用

### 8.5.3 出口対策
- 出力内容のガードレール検証
- 有害コンテンツフィルタリング
- 出力ログの保存

## 8.6 AIエージェント固有のセキュリティ要件

| 要件 | 内容 |
|-----|------|
| 外部ツール接続の認証 | OAuth2.0、APIキーの安全な管理 |
| 実行範囲の制限 | 許可されたアクションのホワイトリスト化 |
| タイムアウト設定 | 無限ループ防止、最大実行時間の設定 |
| サンドボックス実行 | 本番環境への影響隔離 |
| 監査ログ | 全外部呼び出しの記録 |

## 8.7 RAG固有のセキュリティ要件

| 要件 | 内容 |
|-----|------|
| アクセス権限管理 | ユーザー/ロール別のデータアクセス制限 |
| データ投入時の審査 | 機密レベルの確認、承認フロー |
| 引用元表示 | 出典の明示によるトレーサビリティ確保 |
| 定期棚卸し | 古い情報・重複情報の削除 |

> **根拠**: IPA p.70
