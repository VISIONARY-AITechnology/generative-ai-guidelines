# 第5章 データ分類と取り扱い

### 5.1 データ分類基準

| 分類 | 定義 | 例 |
|-----|------|-----|
| **公開情報** | 既に公開されている情報、公開しても問題ない情報 | 公開Webサイトの情報、プレスリリース |
| **社外秘** | 社外への開示を制限する情報 | 社内マニュアル、議事録、企画書 |
| **機密情報** | 漏洩により重大な損害が生じる情報 | 経営情報、未発表製品情報、契約書 |
| **個人情報** | 特定の個人を識別できる情報 | 氏名、住所、メールアドレス、顔写真 |
| **特定個人情報** | マイナンバーを含む個人情報 | マイナンバー、基礎年金番号 |
| **顧客機密** | 顧客から預かった機密情報 | 顧客の技術情報、顧客の個人情報 |

> **根拠**: IPA p.40-41、総務省 p.17-18

### 5.2 入力可否マトリクス

| データ分類 | 許可ツール（オプトアウト済） | 未検証ツール | 自社構築環境 |
|-----------|---------------------------|-------------|-------------|
| 公開情報 | ○ | △（要確認） | ○ |
| 社外秘 | △（条件付き可※1） | × | ○ |
| 機密情報 | ×（例外申請必要） | × | △（要審査） |
| 個人情報 | ×（匿名化後のみ） | × | △（要審査） |
| 特定個人情報 | × | × | × |
| 顧客機密 | ×（顧客許可時のみ） | × | △（顧客許可時） |

**凡例**: ○=可、△=条件付き可、×=禁止

**※1 条件付き可の条件**:
- オプトアウト（学習無効化）設定が有効であること
- ログが記録される設定であること
- データの一部のみを入力し、文脈から特定されないこと

### 5.3 入力前確認事項

1. **データ分類の確認**: 入力しようとするデータは何に分類されるか
2. **ツール設定の確認**: オプトアウト設定は有効か
3. **必要性の確認**: 目的達成に本当に必要なデータか（データ最小化）
4. **匿名化の検討**: 個人情報は匿名化できないか
