# 第4章 利用区分

当社における生成AIの利用は、以下の3区分に分類される。区分ごとに適用されるルール・審査レベルが異なる。

> **根拠**: 総務省 p.30-39（AI開発者/提供者/利用者の分類）、IPA 全体

### 4.1 区分1：社内業務での利用

#### 4.1.1 定義
役職員が自身の業務効率化のために生成AIを利用する場合。

#### 4.1.2 対象業務の例
- 文章作成・要約・校正
- コーディング補助（コード生成、デバッグ支援）
- アイデア出し・ブレインストーミング
- 情報検索・調査補助
- 翻訳・多言語対応

#### 4.1.3 適用ルール
- 許可ツールリストに掲載されたサービスのみ使用可
- 第5章「データ分類と取り扱い」に従った入力制限
- 事前申請：**不要**（許可ツール利用時）
- 利用ログ：**自動記録**（ツール側機能）

### 4.2 区分2：顧客データを扱う利用

#### 4.2.1 定義
受託開発等において、顧客から預かったデータを生成AIに入力・処理する場合。

#### 4.2.2 対象業務の例
- 顧客提供データの分析・要約
- 顧客システムのテストデータ生成
- 顧客ドキュメントの翻訳・変換
- RAGによる顧客ナレッジ検索システムの構築

#### 4.2.3 適用ルール
- 顧客との契約でAI利用が許可されていることを確認
- 顧客機密データの入力可否を事前確認
- オプトアウト（学習無効化）設定の確認
- 事前申請：**必要**（AI管理者への申請）
- 利用ログ：**必須保存**（最低1年間）

### 4.3 区分3：顧客へ提供するAI機能・AIエージェント

#### 4.3.1 定義
受託開発・SES等において、顧客向けにAI機能を組み込んだシステムを開発・提供する場合。

#### 4.3.2 対象業務の例
- AI機能を組み込んだWebアプリケーションの開発
- AIエージェント（自動応答Bot、業務自動化等）の開発
- RAGシステムの構築・納品
- AI活用コンサルティング

#### 4.3.3 適用ルール
- AI審査会議での事前審査必須
- 顧客との責任分界の明確化（第10章参照）
- セキュリティ要件・品質要件の適用（第8章・第9章参照）
- 事前申請：**AI審査会議審議**
- 運用後監視：プロジェクト完了後も一定期間のフォロー

### 4.4 区分判定フロー

```
Q1. 顧客へAI機能を提供するか？
    ├─ Yes → 区分3
    └─ No → Q2へ

Q2. 顧客データを入力・処理するか？
    ├─ Yes → 区分2
    └─ No → 区分1
```
