# 第3章 ガバナンス体制

### 3.1 責任者

| 役職 | 責務 | 任命 |
|-----|------|------|
| **AI推進責任者** | 本ガイドラインの統括管理、AI審査会議の議長、重大インシデント時の最終判断 | CTO or 開発部長（代表取締役が任命） |
| **AI管理者** | 日常的な運用管理、問合せ対応、教育実施、ログ監査 | AI推進責任者が任命（兼務可） |

> **根拠**: 総務省 p.20「責任者の明示」、p.27-29「AIガバナンスの構築」

### 3.2 AI審査会議

#### 3.2.1 目的
新規ツール導入、高リスク案件、例外申請等を審議・承認する。

#### 3.2.2 構成
| 役割 | 人数 | 備考 |
|-----|------|------|
| 議長（AI推進責任者） | 1名 | |
| 開発部門代表 | 1〜2名 | |
| 情報セキュリティ担当 | 1名 | |
| 法務担当 | 1名 | 必要に応じて外部顧問参加 |
| 事務局（AI管理者） | 1名 | |

#### 3.2.3 開催頻度
- 定例会議：月1回
- 臨時会議：緊急案件発生時（AI推進責任者の招集による）

### 3.3 意思決定プロセス

```
[利用・導入申請] → [AI管理者による一次審査]
    ↓
[リスク判定]
    ├─ 低リスク → AI管理者承認（即日〜3営業日）
    ├─ 中リスク → 部長承認（5営業日以内）
    └─ 高リスク → AI審査会議審議 → 役員承認（次回定例または臨時）
```

### 3.4 例外承認権限

| リスクレベル | 承認権限者 | 承認期限 |
|------------|-----------|---------|
| 低リスク | AI管理者 | 即日〜3営業日 |
| 中リスク | 部長 | 5営業日以内 |
| 高リスク | 役員（AI審査会議経由） | 10営業日以内 |

> **根拠**: 総務省 p.27-29「アジャイル・ガバナンス」
