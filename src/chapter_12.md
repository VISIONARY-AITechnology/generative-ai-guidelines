# 第12章 事故・インシデント対応

### 12.1 インシデント分類

| 分類 | 例 | 対応レベル |
|-----|-----|-----------|
| 重大 | 機密情報漏洩、大規模サービス停止 | 役員報告、顧客報告 |
| 中程度 | 社外秘情報の誤入力、一時的な誤動作 | AI推進責任者報告 |
| 軽微 | ハルシネーションの検出、軽微な誤出力 | AI管理者報告 |

### 12.2 報告フロー

```
[インシデント発見]
    ↓
[発見者] → 直ちにAI管理者へ報告（口頭・チャット可）
    ↓
[AI管理者] → 初期対応の指示、記録開始
    ↓
[重大度判定]
    ├─ 重大 → AI推進責任者・役員へエスカレーション
    ├─ 中程度 → AI推進責任者へ報告
    └─ 軽微 → AI管理者で対応完結
    ↓
[初動対応] → 被害拡大防止措置
    ↓
[原因調査] → ログ分析、関係者ヒアリング
    ↓
[再発防止策策定] → 対策実施、教育・周知
    ↓
[報告書作成] → 完了報告
```

> **根拠**: IPA p.50-52

### 12.3 機密情報漏洩時の対応

1. **即時対応**: 当該ツールの利用一時停止
2. **影響範囲特定**: 漏洩したデータの特定、影響者の洗い出し
3. **顧客報告**: 顧客データの場合は速やかに顧客へ報告
4. **当局報告**: 個人情報漏洩の場合は個人情報保護委員会へ報告検討
5. **再発防止**: 原因分析、対策実施

### 12.4 再発防止策

- 原因分析（5 Whys、RCA等）
- 対策の策定・実施
- 教育・周知の実施
- ガイドライン・手順の改定（必要に応じて）
