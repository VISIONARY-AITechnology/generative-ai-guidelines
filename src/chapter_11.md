# 第11章 運用ルール

## 11.1 利用申請フロー

### 11.1.1 区分1（社内業務利用）
- 許可ツールリスト内のサービス：**申請不要**
- 新規ツール利用希望：AI管理者に申請

### 11.1.2 区分2（顧客データ取扱い）
1. プロジェクト開始時にAI管理者へ申請
2. 顧客契約のAI利用条項確認
3. データ分類・入力可否の確認
4. 承認後、利用開始

### 11.1.3 区分3（顧客提供開発）
1. 企画段階でAI審査会議に上程
2. リスク評価の実施
3. 顧客との責任分界合意
4. 承認後、開発開始

## 11.2 AI利用台帳

以下の情報を台帳で管理する。

| 管理項目 | 内容 |
|---------|------|
| 利用プロジェクト | プロジェクト名、担当者 |
| 利用ツール | サービス名、バージョン、アカウント |
| 利用区分 | 区分1/2/3 |
| 利用目的 | 具体的な利用目的 |
| データ分類 | 入力するデータの分類 |
| 承認状況 | 承認日、承認者 |
| 利用期間 | 開始日、終了予定日 |

## 11.3 プロンプト・テンプレート管理

- 有用なプロンプトは社内共有リポジトリで管理
- 機密情報を含むプロンプトは共有禁止
- テンプレートの改善提案は随時受付

## 11.4 RAG/ナレッジベース運用

| 運用項目 | ルール |
|---------|-------|
| データ投入 | AI管理者の承認後に投入 |
| データ更新 | 定期的な鮮度確認（四半期ごと） |
| データ削除 | 不要データの速やかな削除 |
| 権限管理 | 参照可能なデータの権限設定 |
| 引用表示 | 出典の明示を必須とする |

> **根拠**: IPA p.46, p.70

## 11.5 モデル更新時の対応

- モデル提供者からの更新通知を監視
- 更新内容の影響評価
- 必要に応じてテスト実施
- 更新履歴の記録
