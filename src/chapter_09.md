# 第9章 品質・安全性要件

## 9.1 ハルシネーション対策

| 対策 | 内容 | 適用区分 |
|-----|------|---------|
| 人間による検証 | 生成物の正確性を人間が確認 | 全区分 |
| RAGの活用 | 信頼できるソースからの情報検索 | 区分2/3 |
| 複数ソースとの照合 | 生成内容を他の情報源で検証 | 全区分 |
| 不確実性の明示 | 「推測である」等の注記 | 全区分 |

> **根拠**: IPA p.41, p.73-74

## 9.2 バイアス対策

| 対策 | 内容 |
|-----|------|
| 出力のモニタリング | 偏りのある出力の検出 |
| 多様な視点の確保 | 複数のプロンプト設計、複数モデルの比較 |
| 人間の判断介在 | 重要な判断には必ず人間の確認を挟む |

> **根拠**: 総務省 p.16-17

## 9.3 Human-in-the-loop

以下の場面では、人間の判断を必須とする。

- 外部公開コンテンツの最終確認
- 顧客への報告・納品物
- 重要な意思決定への活用
- 個人に関する評価・判断

> **根拠**: 総務省 p.17「人間の判断の介在」

## 9.4 再現性の確保

| 対策 | 内容 |
|-----|------|
| プロンプトの記録 | 使用したプロンプトの保存 |
| モデルバージョンの記録 | 使用時点のモデルバージョン記録 |
| 設定の記録 | temperature等のパラメータ記録 |

## 9.5 性能評価（区分3向け）

| 評価項目 | 評価方法 |
|---------|---------|
| 回答の正確性 | テストデータセットでの正答率 |
| バイアスの有無 | 公平性テスト |
| 応答時間 | レイテンシ測定 |
| スケーラビリティ | 負荷テスト |

> **根拠**: IPA p.36
